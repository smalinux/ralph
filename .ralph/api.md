# Task — design only, do not implement

You are in the design phase. Above you have the recent commits, the open issue
files at `issues/`. Read them, then design the API for what the
issues ask for. Do not write, run, or commit implementation code.

Use the `/codebase-design` skill. If this is an existing codebase, explore it
first so the design fits the modules and conventions already in place.

Produce exactly two files.

## `issues/DESIGN.md`
- For each module, the deep, load-bearing interface it exposes — not every small
  function.
- An ASCII diagram of how the modules/APIs relate (dependencies, data flow).
- If extending an existing project: state exactly where the new API plugs into
  the existing one and what it touches.

## `issues/TEST.md`
- Every command needed to build and run the program.
- Every command needed to test each module/API.

Write only these two files.
